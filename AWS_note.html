AWS（Amazon Web Services）

備註：各地區使用上可能有差異，筆記使用US East (N. Virginia) us-east-1

  • 介紹

亞馬遜公司提供的雲計算平台服務，旨在幫助個人、企業和政府機構構建、部署和管理應用程序，以及儲存和處理數據，用戶可以根據其具體需求和預算來配置和使用以下資源，以構建高度可擴展和可靠的應用程序和服務。

  • 記憶體資源

AWS提供了多種虛擬機器（EC2實例），這些實例具有不同的內存配置，用戶可以根據應用程序的要求選擇適合的實例類型。這使用戶能夠有效地管理應用程序的記憶體需求，無需擔心硬體的物理限制。

  • 儲存體資源

  • Amazon S3

對象儲存服務，用於存儲和檢索數據文件。

  • Amazon EBS

Elastic Block Store，提供了虛擬機器的持久性區塊儲存。

  • Amazon RDS

關聯型數據庫服務，用於儲存結構化數據。

  • Amazon DynamoDB

NoSQL數據庫服務，應對大規模、高性能的數據需求。

  • Amazon Glacier

用於長期儲存的低成本儲存服務等。

  • 網路資源

AWS提供了強大的網絡基礎設施，用戶可以根據需要設置虛擬私有雲（VPC）、子網、路由表和安全性組，以構建安全且高性能的網絡架構。此外，AWS還提供了內容分發網絡（CDN）服務，稱為Amazon CloudFront，用於提供低延遲的全球內容傳遞。

EC2（Elastic Compute Cloud）

  • 介紹

  • 虛擬機器（EC2實例）

 EC2實例是虛擬計算機，用戶可以根據自己的需求選擇不同類型的實例，這些實例具有不同的處理器、內存和存儲容量。用戶可以在這些虛擬機器上運行各種應用程序，包括Web應用程序、數據庫、應用伺服器等。

  • 彈性和可擴展性

EC2實例是彈性的，用戶可以根據需要快速啟動、停止、重新啟動和刪除它們。此外，用戶可以輕松地擴展其計算資源，以滿足變化的工作負載需求。

  • 多區域和多可用性區域

EC2實例可以在AWS的不同區域和多個可用性區域中運行。這使得用戶可以實現高可用性和災害恢復，以確保其應用程序持續運行。

  • 操作系統選擇

用戶可以選擇運行各種不同的操作系統，包括Linux和Windows，並根據需要自行設定和管理這些實例。

  • 安全性

EC2實例可以設置安全性組，以控制入站和出站流量，同時用戶還可以使用AWS Identity and Access Management（IAM）來管理對實例的訪問權限。

  • 存儲選擇

EC2實例可以附帶各種存儲選項，包括Amazon EBS（Elastic Block Store）用於區塊儲存、Amazon S3（Simple Storage Service）用於對象儲存，以及其他選項。

  • 監控和自動化

用戶可以使用Amazon CloudWatch來監控EC2實例的性能和運行狀態，同時還可以使用Auto Scaling來自動擴展或縮減實例數量以應對流量變化。

CDN（Content Delivery Network）

  • 特點

全球內容分發： CloudFront 在全球範圍內擁有多個伺服器位置，稱為邊緣位置。這些位置位於不同的地理位置，使得用戶能夠更快速地存取網站內容，因為它們可以從離用戶最近的邊緣位置提供。

內容快取： CloudFront 可以將靜態和動態內容快取到邊緣位置，從而減少對原始伺服器的訪問。這有助於提高網站性能並節省伺服器流量。

安全性： CloudFront 可以集成其他 AWS 服務，如 AWS Identity and Access Management（IAM）和 AWS WAF（Web Application Firewall），提供安全性功能，以防止網站內容遭受攻擊。

SSL/TLS 支持： CloudFront 支持使用 HTTPS（SSL/TLS）加密協定，確保數據在傳輸過程中的安全性。

動態內容加速： 除了靜態內容，CloudFront 也可以加速動態內容，如動態生成的網頁或 API 請求的回應。

即時監控： CloudFront 提供即時監控和報告功能，讓開發者可以追踪內容分發性能、用戶請求和其他重要指標。

"Statement": [

{

"Resource": "arn:aws:s3:::bk-testvpc-qwezxca123/*"

}

]

}

Version: 這個欄位指定了Bucket Policy的語法版本，這裡是"2012-10-17"，表示採用2012年10月17日的版本。

Statement: 這是一個包含授權聲明的陣列。每個聲明都描述了一條存取規則。

Sid: 是這條規則的唯一識別符，這裡是"PublicReadGetObject"。

Effect: 指定了這條規則的作用，這裡是"Allow"，表示允許存取。

Principal: 這裡設置為"*"，表示這條規則應用於所有實體，即允許所有主體（匿名和已驗證的使用者）。

Action: 指定了被允許的操作，這裡是"s3:GetObject"，表示允許對S3 Bucket中的對象進行讀取操作。

Resource: 指定了這條規則應用的資源，這裡是"arn:aws:s3:::blog.smallko.online/*"，表示允許對指定Bucket（blog.smallko.online）下的所有對象進行讀取操作。
